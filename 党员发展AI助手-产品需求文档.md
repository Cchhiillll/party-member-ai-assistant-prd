## 1. 产品概述

### 1.1 产品定位

面向党务工作者与基层党组织的**党员发展AI助手**，通过 AI 技术提升党员发展全流程效率，降低党务工作者工作负担。

### 1.2 核心价值

| 价值点 | 说明 |
|-------|------|
| **效率提升** | 党员活动线上准备时间减少 50% |
| **可控生成** | AI 内容有来源、可复核 |
| **智能引导** | 更智能化、更便捷的办事引导 |

### 1.3 产品愿景

成为基层党组织数字化转型的核心工具，让党员发展工作更高效、更规范。

## 2. 用户分析

### 2.1 目标用户

| 用户类型 | 角色 | 核心需求 |
|---------|------|---------|
| **党务管理员** | 支部书记/组织委员 | 高效管理流程、减少重复工作 |
| **发展对象** | 群众/积极分子/预备党员 | 清晰指引、便捷办事 |
| **督导人员** | 上级督导/纪检（暂未覆盖） | 监督检查、数据查阅 |

### 2.2 用户画像

#### 用户1：党务管理员（主要用户）

| 属性 | 描述 |
|-----|------|
| **角色** | 支部书记/组织委员 |
| **年龄** | 30-50岁 |
| **技术水平** | 基础电脑操作能力 |
| **痛点** | 审核效率低、重复填写多 |
| **期望** | 减少繁琐工作、提高效率 |

#### 用户2：发展对象

| 属性 | 描述 |
|-----|------|
| **角色** | 申请人/积极分子/预备党员 |
| **年龄** | 20-35岁 |
| **技术水平** | 熟练使用手机/电脑 |
| **痛点** | 流程不熟、找不到入口 |
| **期望** | 清晰指引、及时提醒 |

### 2.3 用户旅程

**发展对象视角：**
```
提交申请书 → 等待审核 → 参加谈话 → 确认积极分子 → 
培养考察 → 确认发展对象 → 政审 → 确认预备党员 → 
预备期考察 → 确认正式党员 → 归档
```

**党务管理员视角：**
```
审核申请书 → 安排谈话 → 录入记录 → 组织支委会 → 
填报信息 → 组织党员大会 → 建档 → 归档
```

### 2.4 痛点分析

| 用户 | 痛点 | 影响 |
|-----|------|------|
| 党务管理员 | 需在多处重复填写活动信息 | 效率低下 |
| 党务管理员 | 入党申请书审核效率低 | 耗时长、易出错 |
| 发展对象 | 入党申请书模板学习成本高 | 不会写、写不好 |
| 发展对象 | 不熟悉党员发展流程 | 不知道下一步做什么 |
| 发展对象 | 找不到办事入口 | 反复询问 |

## 3. 功能需求

### 3.1 功能总览

| 优先级 | 功能模块 | 描述 |
|-------|---------|------|
| P0 | PC端AI助手 | 办事引导、智能问答 |
| P0 | 申请书审核 | 入党申请书 AI 内容审核及校验 |
| P0 | 谈话记录生成 | 基于文档/截图生成结构化记录 |
| P1 | 消息提醒 | 基于画像与流程的定制化提醒 |
| P1 | 活动管理 | 支委会/党员大会活动辅助 |

### 3.2 功能详细说明

#### 3.2.1 PC端AI助手（P0）

**功能描述**

为党务工作者和申请人提供智能办事引导，回答党员发展流程相关问题，指引用户完成各项操作。

**功能要点**

| 功能点 | 描述 |
|-------|------|
| 流程查询 | 回答"我现在该做什么"等流程问题 |
| 政策解答 | 解答入党条件、材料要求等政策问题 |
| 操作指引 | 指引用户找到正确的办事入口 |
| 材料清单 | 自动生成当前阶段所需材料清单 |

**交互示例**

```
用户: 我刚提交了入党申请书，接下来要做什么？

AI助手: 您好！提交入党申请书后，接下来的流程是：
1. 等待党办接口人审批您的申请书（预计3-5个工作日）
2. 审批通过后，支部接口人会与您进行入党谈话
3. 谈话结束后，支部接口人会录入谈话记录
4. 后续会通知您是否进入积极分子培养阶段

💡 提醒：您可以随时在这里询问流程进展
```

**验收标准**

| 指标 | 标准 |
|-----|------|
| 问题准确率 | ≥ 90% |
| 响应时间 | ≤ 3秒 |
| 用户满意度 | ≥ 4.0/5.0 |

---

#### 3.2.2 入党申请书AI审核（P0）

**功能描述**

自动审核入党申请书内容完整性，校验格式规范性，识别敏感/违规内容，生成审核意见与修改建议。

**审核维度**

| 维度 | 检查项 | 权重 |
|-----|-------|------|
| 完整性 | 入党动机、个人经历、对党的认识、表态内容 | 30% |
| 规范性 | 格式、字数、段落结构 | 20% |
| 真实性 | 内容真实性、个人特色、避免模板化 | 25% |
| 正确性 | 政治立场、表述准确、无敏感内容 | 25% |

**功能要点**

| 功能点 | 描述 |
|-------|------|
| 完整性检查 | 检查必填内容是否完整 |
| 格式校验 | 检查格式是否规范 |
| 内容审核 | 识别政治敏感内容、逻辑问题 |
| 相似度检测 | 检测是否存在抄袭/模板化痕迹 |
| 修改建议 | 生成具体修改建议 |

**输出示例**

```
📋 审核结果：需修改

✅ 通过项：
- 格式规范，包含标题、正文、落款
- 入党动机表述清晰

⚠️ 待改进：
1. 【完整性】缺少对个人成长经历的描述
   建议：补充学习、工作经历及思想转变过程
   
2. 【真实性】内容与模板相似度较高(68%)
   建议：增加个人真实经历和感悟

综合评分：72/100
建议：修改后重新提交
```

**验收标准**

| 指标 | 标准 |
|-----|------|
| 审核准确率 | ≥ 85% |
| 误报率 | ≤ 10% |
| 漏报率 | ≤ 5% |
| 审核速度 | ≤ 10秒/份 |

#### 3.2.3 谈话记录结构化生成（P0）

**功能描述**

基于谈话记录文档（Word/PDF）或截图，自动提取关键信息，生成标准化的结构化谈话记录。

**功能要点**

| 功能点 | 描述 |
|-------|------|
| 文档解析 | 支持 Word、PDF、图片等格式 |
| 信息提取 | 自动提取谈话时间、地点、参与人、内容要点 |
| 结构化输出 | 生成标准谈话记录表格 |
| 内容校验 | 检查必填项是否完整 |

**输出格式**

| 字段 | 内容 |
|-----|------|
| 谈话时间 | 自动提取 |
| 谈话地点 | 自动提取 |
| 谈话人 | 自动提取 |
| 被谈话人 | 自动提取 |
| 谈话主题 | 自动提取 |
| 主要内容 | 结构化提取 |
| 总体评价 | 自动生成 |

**验收标准**

| 指标 | 标准 |
|-----|------|
| 信息提取准确率 | ≥ 90% |
| 格式规范率 | 100% |
| 处理速度 | ≤ 30秒/份 |

---

#### 3.2.4 定制化消息提醒（P1）

**功能描述**

基于党员画像（身份、阶段、角色）和流程进展，发送个性化、及时的消息提醒。

**提醒类型**

| 场景 | 触发条件 | 提醒对象 |
|-----|---------|---------|
| 申请书待审核 | 申请书提交后24h未审核 | 党办接口人 |
| 谈话待安排 | 申请书审批通过3天未安排 | 支部接口人 |
| 预备期满 | 转正前30天 | 预备党员 |
| 会议通知 | 会议前1天 | 相关人员 |

**支持渠道**

- 站内消息
- 微信/短信（可选）
- 邮件（可选）

**验收标准**

| 指标 | 标准 |
|-----|------|
| 提醒及时性 | 触发后5分钟内发出 |
| 提醒准确率 | ≥ 95% |

| 材料关联 | 自动关联所需会议材料 |

**验收标准**

| 指标 | 标准 |
|-----|------|
| 简介生成质量 | ≥ 80%用户采纳 |
| 匹配准确率 | ≥ 85% |
